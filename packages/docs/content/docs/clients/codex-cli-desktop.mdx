Use this guide for [Codex](https://openai.com/codex/) in the CLI or desktop app.

## Install Codex

Follow the current install or sign-in flow from the [Codex docs](https://developers.openai.com/codex/cli), then open the same local workspace you want Bardo to manage.

## Connect Bardo

From the workspace root:

```bash
bardo init --rulebook ./RULEBOOK.md
bardo validate
bardo connect --client codex
```

Expected result: Bardo writes `.codex/config.toml` in the workspace root.

## What Bardo writes

Bardo writes the workspace MCP config for Codex automatically, so normal setup does not require manual editing. Codex then talks to the local Bardo MCP server from the same workspace instead of relying on a hosted gameplay API.

## CLI and desktop app guidance

Start Codex in the same workspace where you ran `bardo connect --client codex`. Use that same workspace in the desktop app too.

## Use Bardo well in Codex

Troubleshooting:
- Restart Codex after connecting if the Bardo integration does not show up right away.
- If play feels too thin, add better campaign notes and run `bardo init --rulebook ./RULEBOOK.md` again.
- If `.bardo/` is missing, re-run `bardo init --rulebook ./RULEBOOK.md` before asking Codex to continue play.
