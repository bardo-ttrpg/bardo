Connect your AI client after the workspace is prepared.

```sh
bardo init
bardo connect --client codex
```

Supported first-class targets are Codex, Codex CLI, Claude Code, OpenCode, Gemini CLI, and Cursor. Bardo writes local MCP configuration that starts:

```sh
bardo-mcp mcp serve --workspace-root .
```

The client talks to Bardo over stdio and reads only the configured local workspace.

Generated local configs should not contain bridge URLs, runtime-status URLs, bearer tokens, API keys, or hosted auth headers. If one does, regenerate it with `bardo connect --client <client>`.
