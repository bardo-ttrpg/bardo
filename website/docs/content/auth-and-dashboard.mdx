# Auth and Dashboard

The website uses Clerk for authentication.

## Routes

- `/sign-in`
- `/sign-up`
- `/dashboard` (protected)

## Protection model

Unauthenticated users requesting `/dashboard` are redirected to `/sign-in`.

## Session UX

- Signed out users see sign-in/sign-up actions in the nav
- Signed in users see account controls and log-out actions
