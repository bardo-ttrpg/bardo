# What is Bardo?

Bardo is an **MCP toolbox** that converts AI agents into capable, persistent game masters for any TTRPG system.

AI agents are powerful narrators — but they forget. Chat context is ephemeral. The moment you close a session, your campaign history is gone.

Bardo solves this by keeping all campaign truth in **your local workspace**: plain markdown files that agents read and write through a structured set of MCP tools.

<Callout type="info">
  Bardo is system-agnostic. It works with D&D 5e, Pathfinder, Blades in the Dark, Call of Cthulhu, or any homebrew ruleset — because the rules live in your files, not in Bardo itself.
</Callout>

## Why it works

| Problem | Bardo's answer |
|---|---|
| Agents forget campaign history | World state lives in markdown files |
| Context windows overflow | Only relevant files are loaded per turn |
| Sessions can't be resumed | State is persisted between every tool call |
| Changing AI models breaks continuity | Model-agnostic — any MCP client works |
| No version control over fiction | Workspace is a plain git repo |

## Core capabilities

- **Persistent state** — campaign data survives session restarts and model switches
- **World sync** — proper names, entities, and locations are written to files the moment they're introduced
- **System-agnostic GM** — the agent follows your markdown rules, not a locked schema
- **Structured dice & checks** — reproducible resolution calls with logged outcomes
- **Resumable sessions** — pick up any campaign from exactly where you left off
- **Markdown-first** — human-readable, git-friendly, portable between tools
