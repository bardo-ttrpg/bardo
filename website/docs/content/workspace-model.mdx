# Workspace Model

Bardo treats your local filesystem as campaign truth.

## Core idea

Every durable campaign artifact is represented in markdown files under a controlled directory structure.

## Typical domains

- `_settings`: preferences and setup metadata
- `world`: locations, scenes, maps, and lore
- `party`: player party state
- `entities`: NPCs and notable actors
- `quests`: objective and progress records
- `state`: current world state + history timeline

## Benefits

- Human-readable data
- Git-friendly versioning
- Portable between tools and agents
- System-agnostic gameplay orchestration
