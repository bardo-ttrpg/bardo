# Getting Started

This guide walks you through installing the Bardo MCP server, preparing a campaign workspace, and running your first session.

<Callout type="warning">
  **Prerequisites:** You need [Bun](https://bun.sh) v1.1+ installed and an MCP-capable AI client such as Claude Code, Cursor, or Cline.
</Callout>

## Installation

<Steps>

### Install the Bardo MCP server

```bash
bunx @bardohq/bardo install
```

This installs the MCP server globally and registers it with your system. Alternatively, run it directly without installing:

```bash
bunx @bardohq/bardo
```

### Create a campaign workspace

Run the initializer inside a new directory to scaffold the expected workspace layout:

```bash
mkdir my-campaign && cd my-campaign
bunx @bardohq/bardo init
```

The scaffolder creates the core domain directories and an empty `_settings/config.md` file.

### Connect your MCP client

Point your AI client at the running Bardo server. See [Agent Clients](/mpc-docs/agent-clients) for per-client configuration snippets.

### Run the setup tool

In your AI client, ask the agent to call the `init` tool. This validates the workspace and writes the initial campaign state:

```
Call the init tool with workspace path: /path/to/my-campaign
```

### Start your first session

With the workspace initialized, use `player_action` for each turn:

```
The party enters the ruined tower. Call player_action with:
  workspace: /path/to/my-campaign
  action: "Investigate the ground floor"
```

</Steps>

## Core tool loop

Every campaign session follows the same pattern:

```
init → player_action → world_sync → player_action → ...
```

| Tool | When to call it |
|---|---|
| `init` | Once at the start of every new session |
| `player_action` | Each time players take a meaningful action |
| `world_sync` | After narration introduces new proper names or entities |
| `state_get` / `state_set` | For direct state inspection or override |
| `markdown_read` / `markdown_upsert` | For reading or writing any workspace file |

<Callout>
  You don't need to call `world_sync` manually in most cases — a well-prompted agent will call it automatically when new entities appear in narration.
</Callout>

## Environment variables

Bardo reads the following optional environment variables:

```bash filename=".env"
# Default workspace path (overridable per tool call)
BARDO_WORKSPACE=/path/to/campaigns

# Port for the MCP HTTP server (default: 3001)
BARDO_PORT=3001

# Log level: silent | info | debug
BARDO_LOG_LEVEL=info
```
