Bardo keeps the public docs intentionally narrow. This section exists to help you install the bridge, connect a client, understand the product boundary, and find billing context without mixing those concerns into every route.

## What Bardo covers

- Install the local bridge with the published shell scripts.
- Connect an MCP-capable client to your local workspace.
- Understand where campaign truth lives and where it does not.
- Check how credits and billing appear in the protected dashboard.

## Recommended path

1. Install the bridge on the machine where your campaign files live.
2. Connect your preferred AI client and approve the browser session.
3. Use the campaign truth model as the mental boundary for local versus hosted behavior.
4. Open the billing page only when you need subscription or usage details.

## What stays outside docs

The dashboard, Clerk authentication flows, bridge approval polling, and API routes stay server-backed. The docs remain static on purpose so they load fast and stay easy to maintain.
