# Plain answers for the essential flows.

Bardo keeps the public site narrow on purpose. If a page does not help someone install the bridge, connect a client, understand the product boundary, or manage account access, it does not belong here.

The docs are local MDX content so they stay fast, static, and easy to edit without pulling UI concerns into every route file.

## What you can do here

- Install the bridge with the published shell scripts.
- Connect an MCP-capable client to the local workflow.
- Understand where campaign truth lives.
- Check how credits and billing relate to the protected dashboard.

## What stays outside the docs

The protected dashboard, Clerk authentication, bridge approval polling, and all API routes stay server-backed. The docs stay static.
