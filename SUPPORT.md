# Support

Use the right channel so the project stays easy to follow.

- Bugs and reproducible problems: open a GitHub Issue.
- Feature requests and docs improvements: open a GitHub Issue.
- Questions, ideas, playtesting stories, and showcases: use GitHub Discussions.
- Security vulnerabilities: follow [SECURITY.md](SECURITY.md), not public issues.

Please include enough context for someone else to reproduce or understand the problem. Avoid sharing private campaign data, secrets, tokens, private URLs, or customer information.
